# 高面片数 3D 场景光照与辐射场可微烘焙方案 (V1.0)

## 一、 背景与需求分析 (Background & Requirements)

* **场景特征：** 原始场景模型具有超高面片数（百万甚至千万级别），包含复杂的全局光照（GI）与材质高光细节。
* **传统方案痛点：** 现有的 3D 工具（如 Blender Cycles, Substance Painter）在将高模细节烘焙至减模后的低模时，由于微观几何的差异，光线追踪会产生硬阴影错位、高光撕裂及严重的噪点/伪影。
* **核心诉求：** 采用可微渲染（Differentiable Rendering）替代传统光线追踪烘焙。即在离线工具中输出高模的多视角真实渲染图作为 Ground Truth (GT)，利用 PyTorch 和光栅化器反向优化出绑定在低模上的光照贴图。
* **技术选型进阶：** 为解决传统静态光照贴图无法表达“视角相关高光”的缺陷，方案引入**球谐函数（Spherical Harmonics, SH）**，将优化目标从单一 RGB 纹理升级为基于网格代理的 2 阶表面辐射场（Surface Radiance Field）。

---

## 二、 核心管线与算法实现 (Algorithm & Pipeline)

技术栈定位为 `Python` + `PyTorch` + `nvdiffrast`。

### 1. 数据制备 (Data Preparation)

* **相机采样策略：** 必须采用半球面均匀分布（Hemisphere Sampling）导出序列帧。若仅沿固定高度环绕拍摄，低模的顶部/底部区域将因缺乏观测（Unseen Regions）产生严重的随机噪声或伪影。
* **矩阵对齐：** 导出相机外参时，严格将 Blender（Z-up，右手）转换为 OpenGL 标准（Y-up，右手，视线指向 -Z），生成精确的 $4 \times 4$ 模型-视图-投影矩阵（MVP）。
* **几何代理提取：** 导出减模低模（`.obj` 或 `.glb`），确保 UV 完全展平且无重叠。

### 2. 参数结构与初始化

* **几何常量：** 顶点位置 `vertices` $[N, 3]$，面索引 `faces` $[M, 3]$，纹理坐标 `uvs` $[K, 2]$，UV 面索引 `uv_idx` $[M, 3]$。
* **可优化目标：** 定义 $[1, H, W, 27]$ 的 `torch.nn.Parameter` 作为 2 阶 SH 纹理（9 系数 $\times$ 3 通道）。
* **初始化逻辑：** 0 阶直流分量（前 3 通道）初始化为 $0.5$（或低模预计算的反照率），其余 24 个高频分量全部初始化为 $0$。

### 3. 前向可微管线伪代码

```python
# 1. 坐标变换与光栅化
vertices_clip = transform_pos(vertices, mvp)
rast, _ = dr.rasterize(glctx, vertices_clip, faces, resolution=[h, w])

# 2. 属性插值与世界坐标构建
tex_coords, _ = dr.interpolate(uvs, rast, uv_idx)
world_pos, _ = dr.interpolate(vertices_world, rast, faces)

# 3. 计算单位化视点方向 (从表面指向相机)
view_dir = normalize(camera_pos - world_pos)

# 4. 纹理采样与 SH 解码
sampled_sh = dr.texture(sh_parameter, tex_coords, filter_mode='linear') # [1, h, w, 27]
sh_basis = eval_sh_basis_27(view_dir) # [1, h, w, 9]

# 5. 颜色重建与背景掩码
rendered_rgb = sum_sh_components(sampled_sh, sh_basis)
mask = (rast[..., 3] > 0).float()
output = rendered_rgb * mask

```

### 4. 损失函数计算 (Loss Formulation)

* **联合重建损失：** $L_{total} = \lambda_1 L_1(I_{render}, I_{gt}) + \lambda_2 (1 - \text{SSIM}(I_{render}, I_{gt}))$。
* **几何偏差掩码：** 若低模法线与高模差异极大（如 $>30^\circ$），对该像素的 Loss 赋予 $0$ 权重，防止几何错位造成的错误梯度污染纹理。

## 三、 工程盲点与深度优化建议 (Blind Spots & Optimizations)

在实际工程落地中，以下维度是极易导致训练崩溃或性能不达标的核心节点：

### 1. 梯度爆炸与显存控制 (Memory & Stability)

* **通道拆解：** 27 通道的 4K 贴图在使用 Adam 优化器时显存占用极高。建议将目标 Tensor 拆分为 9 张独立的 $[1, H, W, 3]$ RGB 贴图，分批调用 `dr.texture`。
* **环境依赖冲突：** `nvdiffrast` 对 CUDA 与 C++ 编译器版本高度敏感。强烈建议弃用 Anaconda，采用 Miniforge 结合 mamba 进行纯净的虚拟环境构建，可有效规避底层驱动与 PyTorch 的 C++ ABI 不兼容问题。

### 2. 移动端与硬件适配考量 (Mobile & NPU Target)

* 若最终资产需在移动端或算力受限平台上运行（例如移动 GPU 或 NPU 硬件加速），27 通道的高分辨率纹理采样会造成极端的显存带宽瓶颈。
* **降阶截断策略：** 参考 3DGS 在移动端的优化逻辑，在导出资产时对 SH 阶数进行分离。0 阶分量（Diffuse）保持高分辨率，1 阶和 2 阶分量（Specular）按需缩减分辨率或直接截断，在带宽与高光保真度之间取得平衡。

### 3. 收敛策略 (Coarse-to-Fine Training)

* 直接在 4K 分辨率下优化极易陷入局部极小值，产生噪点斑块。
* 必须引入多分辨率训练金字塔。将训练分为多个阶段，从 $512 \times 512$ 开始收敛大面积的低频光照，再逐步提升至 $1024$、$2048$ 直至 $4096$，配合 TV Loss (Total Variation Regularization) 平滑未观测区域。

### 4. UV 边界伪影处理 (Seam Padding)

* 光栅化插值会在 UV 边界产生黑边。在损失函数之外，需实现一个自定义的 Padding 算子：每过 $N$ 个 Epoch，根据有效的 UV 掩码，将有效像素向外围空白区域进行膨胀（Dilation），确保 Mipmap 采样始终命中安全颜色域。

## 四、 资产导出与落地部署 (Export & Deployment)

标准 `glTF 2.0` 不原生支持高阶 SH 贴图，需采取以下部署策略：

* **方案 A（标准引擎降级）：** 提取第 0 阶分量（前 3 通道），乘以常量系数导出为标准漫反射贴图（Diffuse/Irradiance）。写入 `.glb` 文件的 `emissiveTexture` 节点，关闭所有动态光照，适用于通用 PBR 渲染器。
* **方案 B（原生着色器级集成）：** 抛弃标准材质系统，将低模、拆分的 9 张 PNG 贴图以及 MVP 数据打包为私有格式。在渲染管线最底层编写特定的 Shader 代码直接对这 9 张纹理进行采样与 SH 还原，以此实现跨端的高保真多视角材质表达。
